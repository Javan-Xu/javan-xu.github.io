The Power of Unrecognized

Posted on 2021-11-02 Edited on 2021-11-06
Symbols count in article: 1.1k Reading time ≈ 2 mins.

自從九月底離開前公司後，持續地修改著履歷資料，也陸續地投出了幾份履歷，扣除那些單純在打廣告 (連投遞開啟通知都沒有) 的職缺，難免還是會有被拒絕的狀況，好一點的公司會回覆個婉謝函，但其它大多數都是看了之後就沒下聞了。問說心理不會受到打擊? 那是難免的。

供給與需求

其實會有這樣的狀況我並不意外，因為為了要找到自己覺得較好或較適合的職缺，所以我 104 的履歷一直都是關閉、主動投遞的狀態。公司挑員工跟我們挑公司，本來就是個供需的體現，當你用高的標準去過濾你覺得適合的職缺，當然對方也會用高的標準來過濾適合他們職務的人員，這本來就是個很正常、很公平的狀況。

另一個我比較擔心的是 “年紀” ，我現在也已經四十初頭歲了，如果企業是因為這個因素而不敢用我，那也怪不得他們，只是這樣子真的很無奈啊! 畢竟這又不是努力就可以改變的事情。只能奉勸要轉職的大家，如果您這時候已經超過四十歲的話，真的要做好評估後再行動。

蓄勢待發

過了一個月、下一份工作卻還沒有下聞，這得確對我家庭經濟跟心理上有很大的壓力。現在回想起來，我還是覺得離開是有必要的，因為雖然我從出社會工作到現在都持續在學習新的技術，也一直想把英文學好，但是最近這一年多的時間，是我感覺到這股前進的力量最明顯也最強烈的時候。

以英文的學習來說，雖然說英文還是一樣很爛，但是因為主管要求每天都要寫工作日誌，剛好公司用到英文的機會也多，我從一開始就用英文寫工作日誌，就這樣持續了一年多，不敢說英文就變得多好，但是愈到後頭、文法檢查工具出現的錯誤數量得確變少了很多。

在技術面，那些在開發設備控制系統從無到有之間要學習、應用的技術跟套件當然都是新的收獲，但最讓我有感的是用 Python coding 的速度，這比我之前用過的其它程式語言快上了許多。我自己綜合歸納起來，有可能是 Python 本身語法的特性，跟我短時間內 coding 的頻率，也有可能是 IDE PyCharm 好用的快捷鍵跟外掛工具，也有可能是一直都在 Mac 上進行開發的關係。

但是這些我覺得我正快速成長的 coding、英文跟新事物學習應用能力，在前公司卻不是會受到重視及鼓勵的事情，相反的，它還可能會受到一些限制。所以我才會說，離開是有必要的。

只是現在處於這轉職的關卡，沒辦法如願地應徵上理想中的職缺，對照著那股蓄勢待發的感覺跟持續被否定的次數，內心真的是五味雜陳啊! 也只能安慰自已說 “隨緣” 了，期待看看會不會有個不錯的職缺跟公司能讓我接續這份成長的動能了，屆時我一定會牢記這份被否定的感覺，為那個願意接受我這個中年男人的公司多盡一份心力的。
